# Gmail SMTP 메일 디자인 스펙

## 목표

현재 텍스트 전용인 SMTP 알림 4종을 `생각을 여는 수학`의 따뜻한 오렌지 브랜드 톤에 맞는 HTML 이메일로 개선한다. Gmail에서 읽기 쉽고 모바일에서도 자연스럽게 축소되며, HTML을 표시하지 못하는 환경을 위해 기존 텍스트 본문을 함께 제공한다.

## 대상 메일

1. 수업 신청 접수
2. 수업 그룹 확정
3. 그룹 편성 제외 및 대기 상태 변경
4. 보호자 이메일 인증

## 시각 방향

선택된 방향은 **따뜻한 에디토리얼**이다.

- 브랜드 색상: `#fff7e8`, `#ff9f2f`, `#ff8a1f`, `#e86f00`, `#3b2506`
- 분위기: 친근하지만 가볍지 않고, 보호자가 핵심 상태를 빠르게 파악할 수 있는 구성
- 공통 상단: 오렌지 포인트 라인, `생각을 여는 수학` 브랜드명, 메일별 큰 제목과 짧은 설명
- 공통 본문: 보호자·어린이 이름을 포함한 개인화 안내와 핵심 정보를 담은 강조 박스
- 공통 하단: `놀이에서 사고력, 교과까지 이어지는 배움` 소개 문구와 발신 전용 안내
- 예약 관련 메일에는 현재 상세 페이지 URL이 없으므로 동작하지 않는 CTA를 추가하지 않는다.
- 이메일 인증 메일에만 `이메일 인증하기` CTA와 버튼 미작동 시 사용할 원문 URL을 제공한다.

## 구현 구조

`NotificationService`에서 HTML 문자열을 직접 조립하지 않는다. 알림별 데이터로 완성된 HTML을 반환하는 전용 이메일 템플릿 모듈을 `backend/src/notifications/`에 둔다.

- 공통 레이아웃 함수가 브랜드 헤더, 본문 폭, 푸터를 구성한다.
- 알림별 템플릿 함수가 제목, 설명, 정보 박스, 선택적 CTA를 구성한다.
- Gmail 호환성을 위해 테이블 기반 레이아웃과 인라인 CSS를 사용한다.
- 외부 폰트, 외부 이미지, JavaScript에 의존하지 않는다.
- 모든 동적 문자열은 HTML 이스케이프 후 삽입한다.

`NotificationService.sendMail`은 `text`와 `html`을 함께 받아 Nodemailer의 `sendMail`에 전달한다. SMTP 미설정 로그는 현재 규칙대로 제목과 텍스트 본문만 기록하며, 발송 실패는 API 요청 흐름으로 전파하지 않는다.

## 메일별 콘텐츠

### 신청 접수

- 제목: 수업 신청이 잘 접수되었어요
- 개인화: 보호자 이름, 어린이 이름
- 강조 정보: 접수 완료와 그룹 편성 후 이메일 안내

### 그룹 확정

- 제목: 함께 배울 그룹이 정해졌어요
- 개인화: 보호자 이름, 어린이 이름
- 강조 정보: 그룹명과 요일·시작/종료 시간 목록

### 그룹 제외

- 제목: 수업 그룹 편성이 변경되었어요
- 개인화: 보호자 이름, 어린이 이름
- 강조 정보: 기존 그룹명과 대기 상태 변경 안내
- 불안감을 키우지 않는 차분한 문구를 사용한다.

### 이메일 인증

- 제목: 이메일 인증을 완료해 주세요
- 개인화: 가입자 이름
- CTA: 전달받은 검증 URL로 이동하는 `이메일 인증하기` 버튼
- 보조 정보: 본인이 요청하지 않았다면 무시하라는 안내와 원문 URL

## 안전성과 접근성

- 이름, 그룹명, 일정 문자열, URL을 포함한 모든 동적 값은 HTML 이스케이프한다.
- 본문 텍스트 대비를 확보하고 핵심 정보는 색상에만 의존하지 않고 제목과 문구로도 구분한다.
- CTA는 충분한 패딩과 명확한 링크 텍스트를 사용한다.
- 전체 콘텐츠 폭은 약 600px로 제한하고 작은 화면에서는 좌우 여백이 축소되도록 한다.

## 테스트 및 검증

TDD의 RED-GREEN 순서로 진행한다.

1. 4종 메일이 Nodemailer에 `text`와 `html`을 함께 전달하는 실패 테스트를 작성한다.
2. 인증 CTA URL, 그룹 일정, 브랜드명과 핵심 제목이 HTML에 포함되는지 검증한다.
3. 특수문자가 포함된 이름·그룹명·URL이 이스케이프되는지 템플릿 단위로 검증한다.
4. 최소 구현으로 테스트를 통과시킨 뒤 중복되는 레이아웃을 정리한다.
5. `backend`에서 관련 테스트와 전체 `npm test`, 빌드를 실행한다.
6. 최종 `git status`와 diff로 테스트·빌드 생성 파일 및 사용자 변경 충돌 여부를 확인한다.

## 범위 제외

- Gmail 계정 또는 SMTP 자격 증명 변경
- 실제 운영 수신자에게 테스트 메일 발송
- 예약 상세 페이지나 새로운 CTA 목적지 추가
- 외부 이미지 호스팅 및 추적 픽셀
- React Email 등 신규 이메일 프레임워크 도입
