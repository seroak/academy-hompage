# 지역 검색 SEO 랜딩 페이지 설계

## 목표

생각을 여는 수학의 실제 교육 대상과 프로그램을 검색 의도별 공개 페이지로 제공해 `흥덕 유아 수학`, `흥덕 사고력 수학`, `초등 저학년 수학학원` 검색에서 검색엔진이 각 주제를 명확하게 이해하도록 한다.

검색 순위를 보장하거나 키워드를 기계적으로 반복하지 않는다. 현재 학원 정보와 실제 교육과정만 사용하며, 검색 유입 사용자가 교육 내용과 상담 경로를 바로 이해할 수 있게 한다.

## URL과 검색 의도

| URL | 대표 검색 의도 | 주요 내용 |
|---|---|---|
| `/courses/young-children-math` | 흥덕 유아 수학 | 유아가 놀이와 교구로 수학을 시작하는 방식, 플레이팩토·요리수 수학 |
| `/courses/thinking-math` | 흥덕 사고력 수학 | 관찰·비교·설명·문제 해결 중심 사고력 수업, 세 프로그램의 연결 |
| `/courses/elementary-lower-grades` | 초등 저학년 수학학원 | 저학년의 개념·교과·사고력 연결, 씨투엠 중심 학습 흐름 |

## 구조와 컴포넌트

세 페이지는 Next.js App Router 서버 컴포넌트로 만든다. 공통 화면 구조는 하나의 재사용 가능한 랜딩 컴포넌트와 데이터 모듈로 분리한다.

- 데이터 모듈은 slug, 제목, 설명, 검색 키워드, 소개 문구, 특징, 추천 대상, 연계 프로그램을 제공한다.
- 공통 랜딩 컴포넌트는 데이터만 받아 제목, 소개, 특징, 추천 대상, 교육과정 링크, 상담 CTA를 렌더링한다.
- 각 `page.tsx`는 고유 metadata, canonical, Open Graph와 JSON-LD를 서버에서 출력한다.
- 교육과정 페이지에는 세 랜딩 페이지로 연결되는 검색 친화적인 텍스트 링크 영역을 추가한다.
- 메인 페이지의 기존 교육과정 영역에도 세 페이지로 연결되는 링크를 추가하되 현재 시각적 위계를 유지한다.

페이지는 현재 디자인 시스템의 따뜻한 색상, 둥근 카드, 반응형 간격을 재사용한다. 새 애니메이션이나 외부 UI 의존성은 추가하지 않는다.

## 구조화 데이터와 검색엔진 정보

각 페이지에 다음 정보를 제공한다.

- 고유 `title`, `description`, `keywords`
- 절대 canonical URL
- 공통 Open Graph 필드와 페이지별 제목·설명·URL
- 학원 제공 교육과정을 나타내는 `Course` JSON-LD
- 홈, 교육과정, 현재 페이지 경로를 나타내는 `BreadcrumbList` JSON-LD
- 운영 중인 네이버 플레이스를 포함한 실제 연관 채널 `sameAs` 정보

JSON-LD 생성은 `frontend/src/lib/seo.ts`의 순수 함수로 추가해 단위 수준에서 입력과 출력이 명확하도록 한다. 사이트맵의 정적 경로 목록에는 세 URL을 모두 포함하고 실제 콘텐츠 수정일을 알 수 없는 정적 페이지에 요청 시각을 매번 `lastModified`로 기록하는 동작은 이번 범위에서 확장하지 않는다.

네이버가 최신 콘텐츠 피드 제출을 권장하므로 공개 공지를 RSS 2.0 형식의 `/rss.xml`로 제공한다. 피드의 channel과 item URL은 `NEXT_PUBLIC_SITE_URL` 기준 절대 URL을 사용하고, 각 item에는 공지 제목, 공개 본문 전체, 발행일과 고유 URL을 포함한다. XML 특수문자를 이스케이프하며 공지가 없거나 백엔드 조회가 실패하면 유효한 빈 RSS 문서를 반환한다. 네이버 웹마스터도구는 item이 한 개 이상인 RSS만 제출할 수 있으므로 실제 제출은 공개 공지가 존재할 때 진행한다.

루트 metadata의 `alternates.types`에 RSS를 선언해 검색로봇과 RSS 리더가 피드를 발견할 수 있게 한다. robots.txt는 공개 콘텐츠와 정적 리소스의 수집을 허용하고 관리자·인증 콜백·개인정보 페이지만 차단하며, 사이트맵 절대 URL을 계속 제공한다.

## 사용자 흐름

1. 검색 또는 내부 링크로 검색 의도별 랜딩 페이지에 진입한다.
2. 첫 화면에서 지역, 대상 연령, 수업 성격을 확인한다.
3. 특징과 추천 대상을 읽고 관련 전체 교육과정으로 이동할 수 있다.
4. 상담을 원하면 기존 `/apply` 신청 페이지로 이동한다.

개인정보 입력이나 API 호출은 새 페이지에서 수행하지 않는다. `/apply`의 기존 인증·신청 흐름도 변경하지 않는다.

## 오류와 경계 조건

- 세 URL은 정적 라우트로만 제공하므로 잘못된 slug를 동적 처리하지 않는다.
- 환경변수 `NEXT_PUBLIC_SITE_URL`이 운영에서 올바르게 설정돼야 canonical, Open Graph, JSON-LD, 사이트맵이 운영 도메인을 가리킨다.
- 기존 교육과정 데이터와 다른 프로그램명이나 효능을 새로 만들지 않는다.
- 기존 작업 중인 `frontend/src/screens/ApplyPage.tsx`와 `frontend/e2e/specs/apply.spec.ts`는 수정하지 않는다.

## 테스트와 검증

TDD로 다음 동작을 먼저 실패시키고 최소 구현으로 통과시킨다.

- Playwright 공개 스모크 테스트에서 세 내부 링크가 올바른 URL을 가리킨다.
- 각 랜딩 페이지가 고유 제목과 핵심 H1을 서버 렌더링한다.
- 각 페이지가 `Course`와 `BreadcrumbList` JSON-LD를 출력한다.
- `sitemap.xml`에 세 URL이 포함된다.
- `rss.xml`이 RSS 2.0 XML, 절대 URL, 공지 본문 전체와 발행일을 제공한다.
- 루트 HTML이 네이버 소유확인 메타와 RSS 자동 발견 링크를 환경에 맞게 출력한다.
- 공개 페이지가 `noindex`나 `nofollow` 없이 표준 링크로 서로 연결된다.

완료 전 `npx tsc --noEmit`과 관련 Playwright 테스트를 실행한다. E2E 전후 `git status`와 diff를 확인해 생성 파일 및 기존 사용자 변경을 구분한다.
