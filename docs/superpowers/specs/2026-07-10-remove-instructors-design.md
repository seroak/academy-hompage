# 강사 도메인 제거 설계

## 목표

강사 프로필과 강좌의 담당 강사 연결을 데이터베이스, 백엔드 API, 프론트엔드, 시드, 테스트에서 완전히 제거한다.

## 범위

- Prisma `Instructor` 모델과 `Course.instructorId` 관계를 제거하는 새 마이그레이션을 추가한다.
- 시드에서 강사 생성과 강좌의 `instructorId` 값을 제거한다.
- 백엔드의 instructors 모듈, DTO, 컨트롤러·서비스 스펙을 삭제한다.
- 강좌 조회·생성·수정 DTO와 서비스에서 강사 참조를 제거한다.
- 공개 강사진 페이지와 관리자 강사 관리 페이지, 메뉴 및 사이트맵 경로를 삭제한다.
- 프론트 API, Zod 스키마, TanStack Query 훅과 관리자 대시보드의 강사 의존성을 삭제한다.
- 강좌 목록·상세·관리 화면에서 담당 강사 표시와 선택 필드를 삭제한다.
- E2E 목 서버, fixture, 페이지 객체, 시나리오에서 강사 의존성을 삭제한다.
- README의 공개 사이트 소개에서 강사를 제거한다.

## 데이터 처리

새 마이그레이션은 기존 `Course` 레코드의 외래 키 제약과 `instructorId` 열을 먼저 제거하고, 이후 `Instructor` 테이블을 삭제한다. 따라서 기존 강사와 강좌 연결 정보는 복구되지 않는다. 강좌 자체는 유지된다.

## 호환성 및 오류 처리

- `/instructors` API와 페이지는 더 이상 제공하지 않는다.
- 기존 강사 관련 프론트 경로 및 API 호출은 제거한다.
- 강좌 API 응답과 입력 계약에서 `instructor` 및 `instructorId`를 제외한다.

## 검증

- 백엔드 강좌 스펙을 새 계약에 맞춰 실행한다.
- Prisma 클라이언트를 생성하고 마이그레이션 상태를 확인한다.
- 프론트 TypeScript 검사와 Playwright 전체 스위트를 실행한다.
- 저장소의 현재 소스와 문서에서 역사적 마이그레이션을 제외한 강사 도메인 참조가 남지 않았는지 검색한다.
